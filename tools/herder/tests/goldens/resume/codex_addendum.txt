## AGENTS (herder lifecycle)

Provision and manage agent sessions through herder. Do NOT spawn with `hcom <n> claude`, stop with `hcom kill`, or drive raw `herdr agent start/send` — those bypass the registry or write text without submitting it. herder-spawned claude/codex bind to the bus from birth, so once running you message them with `hcom send`.

Credential-authenticated verbs receive the current non-secret credential path in a generation-keyed bus notice after seat completion. If that notice is unavailable, recover only the path with `herder credential path --guid "$HERDER_GUID"`; the environment guid is provenance for this lookup, never authentication authority. Pass the resulting file explicitly as `--credential-file PATH`.

- Spawn:  herder spawn --credential-file PATH --role R --agent <claude|codex|bash|...> --prompt 'short task (or a one-line pointer to a brief file)' [--workspace ID] [--split right|down] [--cwd PATH]
- List:   herder list [--all]                 # default: seated + unseated; --all adds retired + lost
- Wait:   herder wait <guid> [--read]          # boot-settle / post-send check — NOT for watching long work
- Send:   herder send --credential-file PATH <guid> 'msg' # verified delivery (guid/label, drift-proof)
- Cull:   herder cull --credential-file PATH --guid GUID  # close pane + unseat (label stays leased)
- Fork:   herder fork <guid> [--prompt P] [--workspace ID] [--split right|down] # fresh tab by default
- Resume: herder resume <guid> [--cwd PATH] [--workspace ID] [--split right|down] # fresh tab by default

Before reporting DONE, release external resources you opened.

Delivery is verified: "queued" injects at the target's next turn (do NOT resend); "NOT confirmed" means read the pane (`herder wait <guid> --read`) before retrying — a blind resend double-submits.
If unsure about syntax, always run `herder <command> --help` (or `hcom <command> --help`) FIRST. Do not guess.

Codex has no Task/subagent tool. Fan out sub-work by spawning peer sessions with `herder spawn` (above). Spawned agents bind to the hcom bus from birth — coordinate via `hcom send`, check with `herder wait <guid> --read`, reap with `herder cull --guid GUID` when done.
